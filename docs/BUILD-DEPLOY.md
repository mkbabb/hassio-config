# Build & Deploy System

TypeScript → esbuild IIFE → Node-RED function nodes. Incremental compilation with MD5-based change detection and hash-matched deployment.

## Pipeline

```
  src/*.ts  →  esbuild (bundle, IIFE)  →  dist/*.js  →  deploy (flows.json patch)  →  Node-RED
     │              │                         │                    │
  .build-cache.json │                  node-mappings.json    backups/*.json
  (MD5 + deps)      │                  (nodeId matching)     (10 most recent)
                 metafile
              (dependency graph)
```

## Commands

| Command | Purpose |
|---------|---------|
| `npm run build` | Incremental build of changed files + dependents |
| `npm run watch` | Auto-rebuild on file changes |
| `npm run build:deploy` | Build + auto-deploy changed functions |
| `npm run deploy -- src/path/file.ts` | Deploy specific file |
| `npm run deploy:all` | Deploy all mapped functions |
| `npm run deploy -- --dry-run` | Preview without changes |
| `npm run deploy -- --force` | Skip change detection |
| `npm run map` | Generate hash-based mappings |
| `npm run map -- --ai` | With AI reconciliation (requires OPENAI_API_KEY) |
| `npm run clean` | Delete dist/ |

## Build Cache

`.build-cache.json` — per-file MD5 hash + dependency list + timestamp. A file rebuilds when:
1. Output .js missing
2. Source hash changed
3. Any dependency changed
4. Cache entry older than 24 hours

## Mapping System

`node-mappings.json` maps `src/path/file.ts` → Node-RED function node IDs. Generated by:

1. **Hash extraction**: MD5 of normalized code (comments stripped, whitespace collapsed, case-insensitive)
2. **Hash matching**: Compare compiled JS hashes against function node hashes in flows.json
3. **Collision resolution**: Name similarity scoring when multiple files share a hash
4. **AI fallback**: GPT-5 semantic matching at 75%+ confidence

Current stats: 48 exact matches, 0 high, 0 orphaned, 1 unmapped.

## Deploy Modes

**API (primary)**: `POST /flows` with `Node-RED-Deployment-Type: nodes` for hot reload (no restart).

**File (fallback)**: Direct flows.json write + addon restart via HA API. Used when API returns 400 (which it currently does — all deploys fall back to file-based).

## Backup System

`backups/` — timestamped flows.json copies, 10 most recent retained. Auto-created before every deployment.

```bash
npm run deploy -- --list-backups       # List available
npm run deploy -- --restore-backup <f> # Restore specific backup
```

## Adding a New File

1. Create `src/domain/new-file.ts`
2. Create function node in Node-RED UI (or add to flows.json manually)
3. `npm run build` → compiles to `dist/domain/new-file.js`
4. `npm run map` → detects hash match → adds to `node-mappings.json`
5. `npm run deploy -- src/domain/new-file.ts` → patches function node code
